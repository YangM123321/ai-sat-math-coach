# AI SAT Math Coach Development Contract

This repository is a production-quality portfolio project intended for GitHub publication and AI Application Engineer interviews.

For every product level, deliver:

1. A professional Product Requirements Document.
2. A complete Engineering Specification.
3. Production-style, runnable code.
4. An explicit folder structure.
5. A testing and evaluation strategy.
6. Clearly separated future improvements.

Engineering work must favor maintainability, correctness, extensibility, testability, security, observability, and clear documentation. Assumptions and tradeoffs must be explicit. Documentation must distinguish implemented functionality from planned architecture and must never claim capabilities that the code does not provide.

Core technology standards include FastAPI, PostgreSQL, SQLAlchemy 2.x, Alembic, Pydantic v2, Docker, pytest, OpenAPI, structured logging, environment-based configuration, type hints, modular architecture, and automated CI.
